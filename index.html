A driving RP.

On contract 100M+ Visits

Driving RP. Name pending disclosure.

I'm programming a weapon system on a driving RP that's done 100M+ visits. Can't say the name in public yet but happy to talk about it.

Built the weapon system from scratch. It works in first and third person off the same rig instead of needing a separate viewmodel. I went with IKControl over regular animations because the game supports a bunch of different character styles, and animations don't apply the same way to all of them.

There's also a drive-by mechanic. Players can sit out the window of a moving car and shoot. Same IK setup drives the aim.

  • Weapon system that works in 1P and 3P off the same rig
  • Procedural IK aiming that handles different character styles
  • Drive-by mechanic for moving cars

FPS Framework 2026.

A full first-person weapon system I built for a client. Game-ready out of the box. You bring your own models and animations, it handles the rest.

Covers all the weapon types you'd want: automatic rifles, pistols, rocket launchers, grenades, and melee. Sliding mechanic, procedural animations on the weapons themselves, and a bunch of anti-exploit work.

Built on top of Roblox's server-authoritative character, which isn't even fully released yet. The server has the real say on where the character is and what they're doing, so a lot of the common exploits just don't work.

  • Every weapon type covered: full-auto, pistol, launcher, grenade, melee
  • Server-authoritative character + anti-exploit
  • Sliding, procedural weapon animations
